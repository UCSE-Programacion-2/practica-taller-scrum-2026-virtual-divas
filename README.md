# Práctica taller Scrum — Alquiler de vehículos (Programación II)

Repositorio de **práctica** para alumnos: replica la **estructura y el flujo** del repo docente **demo-proyecto-integrador** (caso TurniMed), pero el caso de estudio es la **plataforma de alquiler de vehículos** ([Product Brief](docs/product-brief.md)).

Incluye la misma base operativa que el demo: `docs/` por sprint, `docs/taller/` con guías paso a paso, `.github/` (issues y PR), `scripts/` y hooks de commits, y carpetas `backend/` y `frontend/` como en el integrador MERN.

## GitHub Classroom (grupos de 3)

La cátedra publica este repo como **template** y crea una **Group assignment** con equipos de **exactamente 3** integrantes. Cada grupo recibe **un solo repositorio** compartido al aceptar la invitación.

Pasos para docentes y alumnos: [docs/taller/github-classroom-y-template.md](docs/taller/github-classroom-y-template.md).

## Cómo usar este repo (alumnos)

1. Aceptar la assignment en Classroom y unirse al equipo de 3 (ver guía anterior).
2. Leer el brief: [docs/product-brief.md](docs/product-brief.md).
3. Seguir la guía en orden: [docs/taller/README.md](docs/taller/README.md).
4. Entender la correspondencia con el demo docente: [docs/taller/replicar-demo-docentes.md](docs/taller/replicar-demo-docentes.md).

## Reglas de contribución (alineadas al integrador)

- Ramas y **Pull Requests** hacia `main`.
- PRs con revisión: con **3** integrantes, suele pedirse **al menos 1 aprobación** de otro compañero (la cátedra puede exigir 2 como en el integrador).
- Trabajo trazable con **Issues** y tarjetas en GitHub Projects.
- Commits según [docs/convencional-commits.md](docs/convencional-commits.md); hooks: `node scripts/install-git-hooks.mjs`.
